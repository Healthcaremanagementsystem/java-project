# java-project
Health Care Management system

Introduction:
The system will enable patients to book and manage appointments, doctors to maange their schedules and patient records, and administrators to handle user management, appointment scheduling, and system setting.

Problem Statement:
current challenge consist of;
1. Manual data management
2. Inefficient patient record handling
3.Difficulty in appointment scheduling
4.Limited accessibility

System Components:
1.Client-side (Java )
2.Server-side (Java Servlet/JSP)
3.Database (MySQL/Oracle)
4.API Integration (for payment gateway, messaging, etc.)

Patient Workflow:
* Patient Registration and Authentication;
  New patients register by providing personal details.
  Existing patients log in using their credentials.
* Dashboard;
  After login, patients access their dashboard where they can view appointments, medical 
  records, and send messages.
* Booking Appointments;
  Patients can view available slots and book appointments with doctors.
  Appointment confirmation is sent to the patient.

Doctor Workflow:
* Doctor Authentication;
  Doctors log in using their credentials.
  Multi-factor authentication for added security.
* Manage Schedule:
  Doctors can set their availability and manage appointment slots.
  Option to block or open time slots for appointments.
*  View Appointments:
  Doctors can view their daily, weekly, or monthly appointment schedules.
  Details of each patient’s condition or reason for visit.
* Manage Medical Records:
  Doctors can create or update patient medical records.

Administrator Workflow:
* Admin Authentication;
  Administrators log in using their credentials.
  Role-based access control to manage permissions.
* Manage Users;
  Administrators can add, update, or delete user accounts.
  Assign roles (patient, doctor, admin) and manage permissions.
* Manage Schedules;
  Administrators can manage overall schedules and availability of doctors.
  Handle cancellations or emergency slots.
* Billing and Payments;
  Administrators handle billing for services provided.
  Manage insurance claims and payments from patients.

Database Schema:
- ER Diagram:
  - Patient table
  - Doctor table
  - Appointment table
  - Payment table

Technologies Used:
1.Frontend: 
  HTML, CSS, JavaScript
2.Backend:
  Java
3.Database: 
  MySQL

Benefits of the System:
 - Improved patient care
 - Increased efficiency
 - Enhanced data security
 - Reduced costs
 - Better decision-making
